# PIPELINE UNIFICADO: ACTIVACIÓN DE NICHO → MATCHMAKING → OPTIMIZACIÓN DE DECISIÓN

Este documento detalla el flujo continuo de tres capas diseñado para convertir un nicho activo en decisiones ejecutables con mínima fricción.

---

## MÓDULO INTEGRADO: LA CADENA CAUSAL

El sistema opera como un pipeline continuo donde cada fase es pre-requisito de la siguiente:
1. **Construcción Estructural (Fase 1):** Define el tablero de juego.
2. **Resolución Algorítmica (Fase 2):** Calcula las jugadas óptimas.
3. **Simplificación Cognitiva (Fase 3):** Ejecuta la decisión humana.

---

## FASE 1 — MOTOR DE ACTIVACIÓN DE NICHO (Estructura del mercado)
**Propósito:** Crear el entorno donde el matching sea estructural, no estadístico.

### Procesos Internos
1. **Identificación de Variables Críticas:** Atributos que determinan la compatibilidad real.
2. **Esquema de Datos Modular:** Entidades de demanda, proveedor y variables de resolución.
3. **Taxonomía Jerárquica:** Categoría → Subcategoría → Solución.
4. **Ontología Semántica:** Sinónimos, equivalencias técnicas y reglas de inferencia.
5. **Definición de Pesos:** Establecer la importancia de cada variable para el matching.

---

## FASE 2 — MOTOR DE MATCHMAKING PROBABILÍSTICO (Resolución óptima)
**Propósito:** Encontrar probabilidades de solución, no solo opciones.

### Cálculo de Score Dinámico
`score_total = (Especialización × Peso) + (Reputación × Peso) + (Logística × Peso) + (Velocidad × Peso) + (Compatibilidad × Peso)`

### Procesos Internos
1. **Filtrado de Compatibilidad Dura:** Eliminación de candidatos no resolutivos.
2. **Evaluación Probabilística:** Cálculo de la probabilidad de resolución real.
3. **Normalización y Ranking:** Selección del Top 3 priorizado.

---

## FASE 3 — OPTIMIZADOR DE DECISIÓN DEL USUARIO (Ejecución humana)
**Propósito:** Reducir la sobrecarga cognitiva para convertir coincidencias en acción.

### Procesos Internos
1. **Limitación de Alternativas:** Máximo 3 opciones visibles.
2. **Traducción de Impacto:** Convertir métricas técnicas en beneficios prácticos (tiempo, riesgo, confiabilidad).
3. **Opción Dominante:** Destacar la recomendación estadística principal.
4. **Medición de Claridad:** Asegurar que el usuario tiene información suficiente para decidir.

---

## LÓGICA DE CONTROL (Gates de Calidad)
- **Gate 1 (Estructura → Match):** Requiere esquema completo y taxonomía consistente.
- **Gate 2 (Match → Decisión):** Requiere probabilidad significativa y ranking estable.
- **Gate 3 (Decisión → Ejecución):** Requiere claridad de decisión y opción dominante identificada.

---

## ARQUITECTURA TÉCNICA
Implementado como un pipeline de servicios coordinados por el **Orquestador Supremo**, utilizando un modelo *event-driven* para la transición entre fases.
